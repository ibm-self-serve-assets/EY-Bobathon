1Executive Summary

The Modern Tax Portal has been successfully migrated from an ASP.NET Web API application running on .NET Framework 4.8 to a .NET 8 Minimal API. All 14 HTTP endpoints have been preserved with identical route paths, HTTP methods, and response behaviours.

The full tax domain logic — bracket-based tax calculation, multi-rule risk band classification, server-side validation, audit trail recording, and dashboard aggregation — has been carried across without modification. Framework-specific plumbing was replaced with modern equivalents; business logic files were reused or refactored with structural changes only.

A 63-test integration test suite was written against the migrated application using WebApplicationFactory<Program>. All 63 tests pass.

9Known Risks

Carried forward from legacy (pre-existing)

IDRiskDetailSeverity
K1TaxPayerId not enforcedA filing can reference a non-existent taxpayer; no FK validationMedium
K2TaxYear format unvalidatedAny string accepted — no year-format checkLow
K3FilingType unvalidatedUnknown types silently default to 21% corporate rateMedium
K4RiskBand dualityStored RiskBand (set at create, defaults to "Medium") and computed RiskBand can divergeMedium
K5No TaxPayer create validationOnly a null-body check; no field-level rulesLow
K6No authenticationAll endpoints are publicly accessibleHigh (prod)
K7No loggingNo request/response or structured telemetryMedium

Introduced or changed during migration

IDRiskDetailSeverity
N1JSON serialisation changeNewtonsoft.Json (PascalCase default) → System.Text.Json (camelCase). Consumers parsing PascalCase property names may break.Medium
N2BadRequest body shapeLegacy returns a plain-text string; Minimal API returns the string as a JSON string literal. Error-body parsers may need updating.Low
N3Concurrent write safetySingleton List<T> is not thread-safe under Kestrel's concurrent request model. Latent bug — more exposed than in IIS.Medium
N4Swagger gated to DevelopmentSwagger UI unavailable in production without explicit config change.Low

10Production Hardening Recommendations

Pri.RecommendationRationale
CriticalReplace in-memory store with EF Core + databaseData lost on restart; List<T> not thread-safe. EF Core with SQL Server / PostgreSQL is the natural next step.
CriticalAdd authentication & authorisationAll 14 endpoints are open. Add JWT bearer auth; apply .RequireAuthorization() on write endpoints.
HighAdd concurrency safetyUntil DB: guard all write operations with lock or SemaphoreSlim.
HighMake service methods asyncRequired for non-blocking I/O once a real database is in place.
HighAdd structured loggingInject ILogger<T>; ship to Application Insights, Seq, or equivalent.

MediumAdd DataAnnotations / FluentValidationReplace manual validation; add rules to TaxPayer create and document upload.
MediumConfigure CORS policyAdd builder.Services.AddCors() for browser-based consumers.
MediumEnforce referential integrityValidate TaxPayerId on filing creation.
LowValidate FilingType and TaxYear formatAdd allowed-value checks to prevent silent calculation fallthrough.
